§ 01 · mathematical engine

The old rule survives. Only the world changes.

The important conceptual move is that 3D phyllotaxis does not require a totally new law. We keep the original sequencing rule: each new point turns by a fixed divergence angle, and the normalised radius grows with the square root of its index. What changes is the surface on which that sequence is allowed to live.

φ = (1 + √5) / 2
α = 2π(1 − 1/φ) = 2π / φ² ≈ 2.399963 rad ≈ 137.507764°
uₙ = (n + 1/2) / N     ρₙ = √uₙ     θₙ = nα
// u is age/progress, ρ is normalised radial growth, θ is angular memory.

In the flat sunflower, the embedding is simply a disc. In the 3D version, the same sequence is passed through a surface function. This is why the piece can become a dome, sphere, torus, cone, shell, or canopy without changing its underlying arithmetic.

flat disc:    Pₙ = (Rρₙ cos θₙ, Rρₙ sin θₙ)
3D body:     Pₙ = F(ρₙ, θₙ, uₙ, t) = (xₙ, yₙ, zₙ)
// 3D phyllotaxis is a parameterised embedding of a 1D growth sequence into a 3D body.
§ 02 · surface grammar

Each mode is a different interpretation of growth.

The dropdown menu is not a collection of decorative presets. It is a small grammar of possible bodies. Each mode asks: what should radius mean? What should height mean? Is the plant growing outward, upward, around, inward, or through time?

§ 03 · why √u matters

The square root is the quiet density corrector.

The square-root radius is not an aesthetic trick. In a disc, area grows with radius squared: A = πr². If every new seed is meant to occupy roughly equal area, then the radius must grow with the square root of the seed index. This prevents overcrowding at the centre and thinning at the edge.

Wrong intuition

If r grows linearly with n, the early points are pushed outward too aggressively and the centre becomes visually distorted.

rₙ ∝ n

Area-corrected growth

If equal index increments correspond to equal area increments, then r² grows linearly, so r grows as √n.

πr² ∝ n → rₙ ∝ √n

In the 3D prototype, this remains useful even when the body is not flat. The variable ρ behaves like a stable growth coordinate: it says where the point sits in the organism’s history, before each mode reinterprets that history as height, latitude, curvature, or time.

§ 04 · programming anatomy

The code is a small rendering pipeline.

The program follows a simple pipeline: generate points from the phyllotactic sequence, map them onto a 3D surface, rotate the virtual object, project it onto the 2D canvas, depth-sort it, and draw the points from back to front.

Generate. For each index n, compute u, ρ, and θ. This gives each point a place in the sequence.
Embed. The function pointFor(i,p) converts the abstract sequence into x, y, z using the selected growth body.
Rotate. The drag interaction changes two angles, rotX and rotY, producing a virtual camera orbit.
Project. Perspective projection makes farther points smaller and nearer points larger.
Sort and draw. Points are ordered by depth, then painted back-to-front so the body reads spatially.
for each seed i: u = (i + 0.5) / N ρ = sqrt(u) θ = i * goldenAngle + twist(ρ) point3D = surfaceMode(ρ, θ, u, time) point3D = rotateXrotateY(point3D, dragAngles) point2D = perspectiveProject(point3D) sort points by z-depth paint far points first, near points last

The animation is deliberately subtle. A small breathing term perturbs the points so the object feels botanical instead of mechanical. This is not biologically exact simulation; it is a visual essay about how a local ordering rule can acquire different bodies.

§ 05 · implications

What changes when phyllotaxis becomes 3D?

The implication is larger than “sunflower, but spatial.” Once phyllotaxis becomes a surface grammar, it becomes a bridge between mathematics, biology, architecture, computation, and systems thinking.

Mathematical

The rule separates sequence from embedding. This is powerful: the same arithmetic order can inhabit multiple geometries, suggesting a family of related forms rather than one canonical diagram.

Biological

It helps us see morphogenesis as local decision-making under constraint. A plant does not need to know φ; repeated local placement can still generate globally elegant order.

Architectural

Phyllotaxis can become a method for distributing apertures, columns, plant modules, acoustic baffles, sensors, or gathering points without imposing a rigid grid.

Computational

This is a compact lesson in generative design: a small set of parameters can generate a large design space, while preserving recognisable structural coherence.

Urban / policy

As metaphor, it shows how a repeated local rule can create emergent global pattern. That is relevant to city form, infrastructure, public-space distribution, and policy terrain thinking.

Pedagogical

It reframes mathematics as transformation rather than notation. Students can feel how a formula becomes a body by rotating, tuning, and comparing outcomes.

2D phyllotaxis explains optimal packing. 3D phyllotaxis asks a deeper question: if growth has memory, what kind of body does that memory choose to become?
